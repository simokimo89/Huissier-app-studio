# ⚖️ منظومة المفوض القضائي الذكية | Smart Judicial Bailiff Platform

منظومة رقمية متكاملة مصممة خصيصًا لرقمنة وتبسيط إجراءات التبليغ والتنفيذ القضائي لمكاتب المفوضين القضائيين بالمغرب، مع حماية السر المهني وتوفير استمرارية العمل الميداني دون الحاجة لشبكة الإنترنت.

A high-performance, secure, offline-first full-stack application engineered specifically for Moroccan judicial bailiffs, sworn clerks, and legal accountants to digitized notification and execution workflows while maintaining absolute professional secrecy.

---

## 🛠️ الركائز الأساسية للمنظومة / Architectural Core

### 1. العمل الميداني دون اتصال (Offline-First Design)
* تم استخدام قواعد بيانات **Dexie.js (IndexedDB)** على المتصفحات لتخزين وسجل البيانات محلياً.
* آلية طابور ذكية (`syncQueue`) تقوم بجدولة العمليات وتتبع التعديلات تزامنياً ليتم رفعها للخادم المركزي فور استعادة الشبكة بلمسة زر واحدة.

### 2. التشفير العسكري لحماية السر المهني (AES-256 Vault)
* امتثالاً للسر المهني المنصوص عليه قانوناً، يتم تشفير المعلومات الحساسة للمتقاضين (اسم المدعي، اسم المدعى عليه، وعنوانه) تلقائياً بالمتصفح بواسطة خوارزمية **AES-256** قبل حفظها بذاكرة الجهاز.
* يتم الحصول على مفتاح الفك والربط محلياً من رمز مرور الولوج الآمن الخاص بالمستخدم النشط ولا يرسل للعموم.

### 3. نظام الصلاحيات والأدوار الثنائي (Role-Based Access Control)
تضم المنظومة ثلاثة أدوار وظيفية متناغمة ومستويات ترخيص تدريجية:
* **المفوض القضائي (Officer) - المستوى الثالث (3)**: الصلاحية الكاملة على السجلات، الإيرادات والودائع، بالإضافة إلى **لوحة إدارة طاقم العمل** لتعديل الأسماء ورموز المرور لحسابات الفريق بما يضمن ضبط الحماية والمهام الإجرائية.
* **محاسبة المكتب (Accountant) - المستوى الثاني (2)**: تتبع المقبوضات المالية، أتعاب التنفيذ والتبليغات، وإصدار الوصولات وتوثيق الودائع بصندوق المحكمة المعنية.
* **المساعد المحلف (Assistant) - المستوى الأول (1)**: استلام التبليغات الميدانية، إعداد محاضر المعاينة، التقاط وتحميل الصور والوسائط التوثيقية كحجية ملموسة.

## 💻 المكونات التقنية / Tech Stack

* **Front-End**: React 19, TypeScript, Tailwind CSS, Lucide Icons, Motion (Animations).
* **Back-End**: Node.js & Express (API Services, Sync Protocol, Profile Management).
* **Bundle & Compile**: Vite & Esbuild CJS compiler pipeline.
* **Local Storage**: Dexie (IndexedDB) & CryptoJS.

## 🚀 التشغيل والتطوير / Quick Start

### 1. تثبيت الحزم والمكتبات الإضافية / Installation
```bash
npm install
```

### 2. تشغيل الوضع التطويري المباشر / Local Development
يقوم الأمر بتشغيل الخادم المحلي وخادم Vite التفاعلي معاً على المنفذ `3000`:
```bash
npm run dev
```

### 3. بناء المنظومة للإنتاج والرفع / Production Bundling
يقوم بضغط ملفات الواجهات وتجميع خادم الـ Node بالاعتماد على `esbuild` في مجلد مستقل جاهز للتشغيل بأعلى كفاءة:
```bash
npm run build
```

### 4. إطلاق المنصة بوضعية التشغيل النهائي / Launch
```bash
npm run start
```

## 🔒 آلية حماية الولوج وإصلاح رموز المرور (Password Auditing)
بصفتك المفوض القضائي (المسؤول الإداري):
1. قم بولوج المنظومة بحساب الأستاذ الخليفي (`elkhalifi`).
2. انتقل إلى تبويب **إعدادات المكتب والفريق**.
3. ستظهر لك لائحة الطاقم الإداري للمكتب؛ يمكنك تعديل اسم أي مستخدم فوراً أو إعادة تعيين رمز المرور الخاص به. 
4. فور التحديث، يسجل التعديل مباشرة بقواعد بيانات السيرفر لضمان المزامنة والأمان التام.
